# 企业风险核查智能体

这是一个可本机运行的轻量级企业风险核查辅助系统，用于展示税务风险核查场景中的大模型辅助能力与本地 Excel 知识库检索能力。当前版本接入 DeepSeek 官方 OpenAI 兼容 API，不接入数据库、登录、持久化历史记录、RAG 或复杂 Agent 编排。

## 功能

- 智能应对：分为“自由提问”和“下发疑点应对”。自由提问返回回答总结、核查方向、应对措施、参考材料及风险提示等结构化辅助建议。
- 下发疑点应对支持上传 `.xlsx`，解析后以企业为索引管理风险点；进入企业详情后可查看风险点、选择单条疑点或企业全部疑点作为背景生成应对建议。建议记录按企业分类，可在线打开或下载保存，下载文本包含所选背景。
- 报告复核：支持上传 `.xlsx` 报告清单，也支持上传 `.docx` Word 完整报告。Excel 清单用于复核“情况说明”质量，复核结果可在本次页面会话内查看、复制和下载保存；Word 报告用于解析“应对任务基本情况”“任务具体情况”和风险点结构，支持全面复核或仅复核当前选中的风险点，全面复核结果和各风险点复核结果会分别保存，并支持查看、复制和下载。
- 知识库检索：将政策法规和历史案例 Excel 放入固定目录后，可在页面中按关键词检索。
- 示例 Excel：`demo_reports.xlsx`，包含 5 条虚构脱敏报告。
- 示例 Word：`mock_word_report.docx`，包含 1 份虚构完整风险应对报告。

## 依赖安装

建议使用 Python 3.10 或更高版本，以下命令用于在linux系统上创建虚拟环境并安装依赖。
若采用其他环境管理方法，只需安装相应依赖即可。

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## 配置 DeepSeek

在根目录下创建.env文件(/tax-risk-demo/.env)

填写：

```env
DEEPSEEK_API_KEY=你的 DeepSeek API Key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-pro
DEEPSEEK_TIMEOUT_SECONDS=60
```

如果未配置 `DEEPSEEK_API_KEY`，页面可以正常打开，调用智能应对或报告复核时会提示需要配置 API Key。

## 配置知识库 Excel

将知识库 Excel 文件放入 `data/knowledge/` 目录：

- 政策法规：`data/knowledge/policies.xlsx`
- 历史案例：`data/knowledge/cases.xlsx`

系统读取每个文件的首个工作表，并检索所有非空单元格。字段名可以先保持灵活。

政策法规字段从左到右应为：`类型`、`种类`、`标题`、`发文字号`、`时效性`、`正文`、`发文时间`。

建议历史案例字段：`案例名称`、`行业`、`风险类型`、`税种`、`核查情况`、`处理结果`、`启示`。

## 启动

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

浏览器访问：

```text
http://127.0.0.1:8000
```

页面路径：

- 智能应对：`/service/smart-response`
- 报告复核：`/service/report-review`
- 政策法规：`/knowledge/policies`
- 历史案例：`/knowledge/cases`

## 最小测试流程

1. 打开首页，确认标题为“企业风险核查智能体”。
2. 访问 `http://127.0.0.1:8000/api/health`，应返回 `{"status":"ok"}`。
3. 在“智能应对”的“自由提问”输入一个风险核查问题，点击“开始分析”。
4. 切换到“下发疑点应对”，上传 `demo_risk_clues.xlsx`，在企业索引中进入某企业详情，选择单条疑点或企业全部疑点后生成应对建议。
5. 在“报告复核”上传 `demo_reports.xlsx`，选择任意一条报告后点击“开始复核”。
6. 切换到“Word 完整报告复核”，上传 `mock_word_report.docx`，可选择“全面复核”或选中某个风险点后“复核当前风险点”。
7. 查看复核结果表格，并点击“复制复核结果”验证剪贴板复制。
8. 将知识库 Excel 放到 `data/knowledge/` 后，在“政策法规”或“历史案例”菜单中输入关键词检索。

## 上传文件格式

### 智能应对下发疑点清单

- 仅支持 `.xlsx`。
- 读取首个工作表。
- 列顺序固定为：`序号`、`纳税人名称`、`疑点名称`、`风险所属期`、`风险描述`。
- 第一行可以有表头，也可以没有表头；如果第一列没有列名，系统仍按第一列作为序号读取。
- 页面不会全量展开所有疑点正文；解析后先展示企业索引，支持按纳税人名称、疑点名称、所属期或风险描述检索企业。
- 进入企业详情后可查看该企业风险点，继续在当前企业内检索风险点，并选择单条疑点或企业全部疑点作为大模型分析背景。
- 生成建议记录时会保留上述全部字段，并按企业分类管理；下载保存的文本会包含背景范围、背景名称和下发疑点清单。
- 上传文件不会保存到服务端磁盘；解析结果和建议记录仅用于当前浏览器会话，下载保存由浏览器生成文本文件。

### 报告复核清单

- 仅支持 `.xlsx`。
- 读取首个工作表。
- 上传文件保持原始导出字段名即可，示例文件不需要改表头；系统会在前端展示为“情况说明”“疑点信息”“人工认定结果”等中文业务名称。
- 如文件包含疑点信息、人工认定结果、申报更正情况等字段，系统会在解析后展示，并在复核时作为风险简述、人工认定结果和更正情况参考。
- “人工认定结果”字段取值应为“有问题”或“无问题”，分别表示工作人员处理后认定风险成立或不成立。
- “风险任务名称”建议采用“年份月份管理条线批次地区税务局批次行业风险任务”格式，例如“2026年06月增值税管理条线第01批上海市税务局第01批软件和信息技术服务行业风险任务”。
- 上传文件不会保存到服务端磁盘；解析结果和复核结果仅用于当前浏览器会话，复核结果可由浏览器下载保存为文本文件。

### Word 完整报告

- 支持 `.docx`；旧版 `.doc` 请另存为 `.docx` 后上传。
- 系统会提取 Word 正文和表格文本，并尽量识别以下结构：
  - `应对任务基本情况`
  - `任务具体情况`
  - `风险点x：xx风险`
  - `风险点具体描述`
  - `验证情况`
  - `拟处理意见`
  - `政策依据`（可选）
- `风险点具体描述` 可以显式写标题；如果未写标题，系统会把 `风险点x：xx风险` 与 `验证情况` 之间的正文识别为风险点具体描述。
- `拟处理意见` 只要包含 `风险排除` 或 `风险确认` 即可，允许带句号或后续说明。
- 解析结果不会全量铺开展示；页面提供风险点检索、紧凑列表、选中详情和全文摘要折叠预览。
- 可对整份 Word 报告全面复核，也可先选中某个风险点后仅复核当前风险点。
- Word 复核结果支持复制和下载；全面复核记录和每个风险点的单点复核记录分开保存，下载文本包含文件名、风险点数量、解析警告、复核范围及当前风险点背景。
示例文件：`mock_word_report.docx`。
